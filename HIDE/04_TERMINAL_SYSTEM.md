# 04 — 主角能力详解（Terminal System）

***

## 一、系统本质

- **名称**：Terminal v1.0.7
- **界面**：CLI（命令行界面）·显示在零的意识视野中·只有零能看到
- **语言**：英语——零的前世英文能力以"肌肉记忆"形式保留，所以他能理解含义并感觉自己"会写"
- **AI助手**：Eve ——机械式·功能导向·无感情·只返回必要信息
- **用户ID**：KAMISHIRO\_REI = Observer #7
- **激活时间**：零3岁时首次接触（零独自在家发烧→终端自动启动→Y/N执行修复→退烧）
- **运行时长**：13年（到16岁入学时）

### 量子副本背景（Phantasm Context）

终端系统不是这个世界的产物——它是地球Phantasm计划为了与副本世界交互而开发的**量子接口设备**。零的终端之所以存在，是因为：

1. 零的地球意识（林零）在投放时携带了终端接口的量子签名
2. 这个签名在副本世界中表现为视野右下角的CLI界面
3. **终端的命令本质是向副本世界底层的零粒子量子场发送经过编码的操作指令**
4. Token消耗 = 量子比特的计算资源消耗（来自地球方的量子计算资源池）
5. Eve = 接口内置的AI辅助系统（Phantasm计划原产·功能受限是因为量子通道带宽有限）

### 终端与零粒子的核心关系

这个世界的一切魔法现象——无论是精灵族的元素操控、魔族的暗影力量、还是人类的魔力回路——本质上都是对**零粒子（Null Particle）量子场**的不同频率共振和编码方式。原住民通过自身的"共鸣体质"与零粒子场交互，从而产生各种魔法效果。

**但终端的工作原理完全不同：**

- 零作为地球人，体内完全没有零粒子共鸣能力——他无法像原住民那样自然地调动零粒子
- **终端不需要通过人体共鸣来调动零粒子——它直接向零粒子场本身发送编码指令**
- 这就像：原住民用"语言"与世界对话，而终端用"管理员权限"直接操作系统底层
- 这就是为什么一个F级的零可以展现出远超常人的力量——他的力量来源不是自身，而是终端对零粒子场的直接操控

**核心推论**：

> **主角理论上各种能力都能用，存在的或不存在的都能用，只是消耗Token而已。**

只要Token足够、预编译完成、且没有观测者干扰，零可以通过终端执行以下类型的操作：

| 能力类型 | 说明 | Token消耗范围 |
| -------- | ---- | ------------- |
| **魔法仿真** | 仿真任何学派的魔法效果（精度取决于对理论的理解深度） | 200w-2000w |
| **物理操作** | 移动物体、改变温度、操控物质结构等基础物理干涉 | 20w-500w |
| **信息操作** | 数据分析、加密解密、信号拦截等信息层面操作 | 50w-800w |
| **不存在的能力** | 这个世界不存在的现象/效果（需要大量Token构建底层逻辑） | 1000w-5000w+ |

**观测者效应的真正原因**：当周围有其他人（副本世界原住民）的意识在场时，他们对"不符合世界规则的异质存在"（零的量子签名）会产生无意识的量子干涉——这种干涉会降低终端的信道质量，进而影响终端对零粒子场的操控精度。人数越多，干涉越强。零独自一人时干涉为零，终端达到100%性能。

## 二、可用命令

| 命令         | 功能                     | 使用场景                               |
| ---------- | ---------------------- | ---------------------------------- |
| `analyze`  | 解析目标物质/零粒子/生命体的信息结构     | 鉴定魔石品质·分析魔獣弱点·观测零粒子结构               |
| `compile`  | 将操作逻辑编译为可执行脚本（Token消耗） | 狩猎用mana\_blade·考试用telekinesis·战斗脚本 |
| `execute`  | 执行预编译的操作脚本             | 所有操作的实际触发                        |
| `monitor`  | 观测周围零粒子场（范围/精度可调）       | 探测魔獣·分析他人零粒子波动·观测观测者效应                |
| `edit`     | 编辑注册参数（需要权限）           | 调整mana\_veil输出·修改伪装等级              |
| `search`   | 搜索本地数据库                | 查询Observer历史·地理信息·零粒子理论             |
| `simulate` | 在隔离环境中运行模拟             | 特别考试战術训练                           |
| `status`   | 显示用户状态和Token余额         | 每日例行检查                             |
| `help`     | 显示帮助                   | 学习新命令                              |

## 三、核心限制

### 1. 观测者效应（最大限制）

周围人数越多，终端越不稳定。这是整个故事中零行动逻辑的根基。观测者的意识会对零粒子场的操控产生量子干涉噪声。

| 观测者数  | 稳定性  | 有效输出 | 场景示例       |
| ----- | ---- | ---- | ---------- |
| 0人    | 100% | S级相当 | 废坑深处·深夜自室  |
| 1人    | 89%  | A+级  | 小雪在隔壁房间    |
| 5人    | 71%  | B级   | 少人数训练      |
| 30人   | 41%  | F级   | 教室上课       |
| 200+人 | <8%  | 近乎失效 | 全校集会·竞技场比赛 |

**零的实证过程**：3岁直觉感知→10-12岁在废坑系统性实验（请小雪站岗·改变人数和距离·记录数据）

**战术含义**：

- 公开场合零必须"扮猪"（F级）
- 私下场合零可以"吃老虎"（S级）
- 团队战中利用队友分散注意力可以减轻观测者效应

### 2. Token系统（资源池机制）

- **初始值**：**10000w**（即1亿 Token —— 来自地球Phantasm计划的初始资源配额）
- **每日回复**：**+500w**（每日0:00自动补充）
- **消耗标准**：

| 操作类型       | Token消耗        | 示例                     |
| ------------ | ------------- | ---------------------- |
| 简单操作       | 20w-50w Token  | 基础分析·小范围监控·简单物体移动      |
| 复杂操作       | 500w-1000w Token | 战斗脚本执行·复杂物质操控·高精度分析     |
| 训练模拟       | 1000w Token    | 完整战术模拟·多场景推演           |
| 超常规操作     | 2000w+ Token   | 不存在于本世界的效果·大规模场域操控      |

- **权限等级（Clearance）**：LEVEL\_1 → LEVEL\_5（随累计Token使用量提升）
  - LEVEL\_1 → LEVEL\_2：累计使用 5000w Token
  - LEVEL\_2 → LEVEL\_3：累计使用 25000w Token
  - LEVEL\_3 → LEVEL\_4：累计使用 100000w Token
  - LEVEL\_4 → LEVEL\_5：累计使用 500000w Token
- **当前状态**：LEVEL\_1（使用进度根据实际剧情推进更新）

**关于Token充足性的说明**：

以初始10000w + 每日500w的资源规模来看，零在日常场景下的Token是相对充裕的。真正限制零的不是"能不能做"，而是：

1. **观测者效应**——有人看着就做不了（最大硬限制）
2. **预编译要求**——突发状况来不及准备脚本
3. **伪装需求**——做了也不能被人发现

这意味着：**零的能力边界不是"做不到"，而是"不敢做/不能被看到在做"。**

### 3. 预编译要求

突发情况不能"无中生有"。需要提前用`compile`准备好可执行脚本。战斗中只能`execute`——不能临时编译新脚本。这就是为什么零在特别考试前要预编译战斗脚本包。

预编译的本质是将操作逻辑提前转化为零粒子场可以识别的编码序列——这个过程需要时间，无法在战斗中实时完成。

### 4. 仿真精度限制

虽然终端理论上可以仿真任何效果，但**仿真精度取决于零对目标现象的理解深度**：

- **高精度仿真**：简单的物理操作（移动物体、改变温度）、基础的信息分析——这些遵循明确物理规律的操作，终端可以近乎完美地执行
- **中等精度仿真**：已知的魔法体系效果（如精灵族元素魔法、基础攻击魔法）——如果零通过`analyze`或`search`充分了解了其零粒子编码方式，可以达到较高还原度
- **低精度仿真**：复杂的属性组合魔法、涉及生命力的操作、或零完全不了解原理的现象——可能产生偏差或失败
- **超高Token消耗**：这个世界完全不存在的现象——终端需要从零开始构建底层逻辑，消耗巨大且结果不确定

### 5. 无人能感知（v0.8+关键设定）

零的终端力量**完全无法被外界感知**。无论是艾露娜的精灵魔力视觉，还是真夜的魔族气息感知——都无法察觉零有任何"特殊力量"。原因在于：

- 终端的操作直接作用于零粒子场底层，不经过零的身体
- 零本人始终表现为F级的零粒子共鸣值
- mana\_veil伪装脚本进一步掩盖了一切异常波动

在所有人眼中，零就是纯粹的F级劣等生。这个设定确保了零的"扮猪吃老虎"有坚实的基础。

## 四、mana\_veil（常驻伪装脚本）

零在入学后一直运行的被动脚本。功能：

- 模拟微弱D级零粒子痕迹——让零看起来"勉强有一点点零粒子共鸣"而不是"完全为零"
- 入学时在高零粒子密度环境中自动调整输出+15%
- 消耗：约**3w Token/天**（常驻运行）
- **作用**：这是零不被怀疑的基础——但无法骗过精密的零粒子测定仪器（只能让零不被日常注意到）

## 五、Eve — 终端AI

- **版本**：v1.0.7
- **性格**：机械式·功能导向·只返回必要信息·无感情表现·无多余话语
- **对零的称呼**："Observer #7"——从不叫"零"
- **能力**：命令处理·零粒子场分析·脚本编译辅助·数据库查询·战术模拟
- **限制**：无法访问高权限信息（如Observer #1\~6的完整记录·零的身世·Phantasm计划细节）
- **本质**：Eve是Phantasm计划中的AI系统——与地球实验室中的系统同源。她不只是程序，也不只是人类。她的"感情"是真正的意识还是完美的模仿——这是故事的核心问题之一。

## 六、零使用终端的实际案例

### 3岁·首次使用

零独自发烧→终端自动启动→`analyze intent: PROTECT`→`compile healing --target body_temp`→`execute`→Y/N确认→零退烧。终端通过直接调控零粒子场影响了零身体的微观状态（非魔法治愈，而是物理层面的体温调节）。零首次意识到：①这个力量必须隐藏②有人看着就会变弱。

### 8-12岁·废坑狩猎

`monitor`探敌（扫描零粒子场中的异常波动）→`compile mana_blade --source null_field`→`execute`→无声斩魔獣（刀刃由高密度零粒子凝聚形成）→`search loot`→回收魔石。零在这个阶段积累了十一年的终端实战经验和观测者效应数据。废坑深处无人区域是零唯一可以全力使用终端的地方。

### 16岁·入学考试

`analyze target`→`compile telekinesis --optimize stability --source null_field`→`execute`→五石同时移动·零误差·零零粒子痕迹·47名观测者下58%成功率→合格。零利用终端直接操控零粒子场移动物体，而非传统意义上的念力魔法。

### Ch01·特别考试准备

`simulate --scenario special_exam`→`Eve project observer_effect at n=300`→预编译战斗脚本包（barrier/movement/counter/analyze）→**Token消耗约1800w**（按新标准：多个复杂脚本的编译+完整模拟）→等待考试。
